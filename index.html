2. Exercise your mental model: Pick one page in your app. Trace the path: what file renders it, what's the route, what components does it use, where does the data come from?

I chose the page that shows the Books. It's rendered from content-manager/src/app/books/page.tsx. In the same path content-manager/src/app/books/page there's the sub-repo called add/ which is the form that is displayed to add Books to the content manager.

Inside the Book repo, there's also the path called [id] which contains the displayed template of the page of the specific item of the book that was added. In this page we can see the details, timeline of the item, picture, and others. Now, since there's no database, the memory is in the computer.

It's using three main components which are the shared ones among all content types (Content, Books, Movies). This ones are Navigation Bar, Categories Input, and Timeline Status (the one displayed in the specific page of each item).
